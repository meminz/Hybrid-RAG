# Evaluation Queries for Hybrid RAG System
#
# Format: query | version | relevant_docs
#
# relevant_docs: semicolon-separated "doc_path:grade" pairs
#   grade 2 = primary answer (the doc that directly answers the query)
#   grade 1 = supplementary answer (related/useful but not the main answer)
#
# doc_path is the relative path from the version folder (e.g. Window-Rules.md)
# The evaluator matches against the document ID containing this path.
#
# Ground truth: A retrieved doc is relevant if its ID contains the doc_path.
# NDCG@K uses graded relevance (grade 2 > grade 1).
# Recall@K = relevant_retrieved_at_K / total_relevant_docs.
#
